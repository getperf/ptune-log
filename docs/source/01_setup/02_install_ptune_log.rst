ptune-log プラグインインストール
==================================

この章では、Obsidian に **ptune-log プラグイン** を導入する方法を説明します。  
ユーザテスト用の手動インストールと、公開後に予定されている標準インストールの 2 種類があります。

マニュアルインストール（ユーザテスト用）
-----------------------------------------

ユーザテストでは、事前に配布された ``ptune-log-0.0.1.zip`` を使用します。

#. GitHub リポジトリのリリースページから zip ファイルを取得  
   `https://github.com/getperf/ptune-log/releases <https://github.com/getperf/ptune-log/releases>`_
#. Obsidian プラグイン保存ディレクトリに展開します。
#. 初期化されたVaultには plugins ディレクトリがまだないため作成します。
#. 画面左下の、「保管庫の管理」を選択し、Vault のリストから、作成した Vault を右クリック > 
   システムのエクスプローラで表示 を選択し、エクスプローラを開きます。
#. `<Valult>/.obsidian` まで移動し、その下に plugins ディレクトリを作成します。
#. 取得した zip ファイルを本フォルダ配下に展開します::

      <Vault>/.obsidian/plugins/ptune-log/

#. 画面左下の、設定 > コミュニティプラグイン を選択し、インストールされたプラグインの右の **リロード** で反映します。
#. 設定画面に **ptune-log** が表示されることを確認し、右側のトグルボタンから、 **有効化** を選択します。

標準インストール（公開後）
---------------------------

公開後は、Obsidian のコミュニティプラグインブラウザから直接インストールできます。

#. 画面左下の、設定 > コミュニティプラグイン を選択し、コミュニティプラグインの **閲覧** を選択します。
#. 検索バーで ``ptune-log`` を入力
#. 一覧から **ptune-log** を選び、**インストール** をクリック
#. インストール後、「有効化」ボタンを押す
#. 設定 > プラグイン > ptune-log の項目が追加されていることを確認

注意点
---------

- ユーザテスト段階では、 **zip 展開して配置する手動インストール** のみ利用可能です。
- 公開後は **Obsidian 内から直接インストール** できるため、手動配置は不要になります。
- プラグインを更新する場合は、リリースページから新しい zip を取得し、既存ファイルを上書きしてください。

セットアップウィザード
=========================

ptune-log では、初めて利用する際に必要なプラグインやテーマの有効化、  
ノート環境の準備をまとめて行うために **セットアップウィザード** を用意しています。  

このウィザードを利用することで、初見の方でも次の準備を漏れなく進められます。

- 必須プラグインの有効化・インストール
- 推奨テーマの設定
- デイリーノートやノート用ディレクトリの初期化

.. note::

   セットアップウィザードは Obsidian のコマンドパレットから実行します。  
   Obsidian 上で ``Ctrl+P`` を押し、 **「ノートの初期セットアップ」** を検索して選択してください。

利用方法の流れ
--------------

1. コマンドパレットから **ノートの初期セットアップ** を実行すると、セットアップウィザードのダイアログが表示されます。  
2. 画面に表示される **①〜⑥ の手順** を上から順に実施してください。  
3. 手順を進めた後は、ダイアログ下部の **「更新」ボタン** をクリックすることで、セットアップ状況を再確認できます。  
4. すべての項目が完了すると、ptune-log を利用する準備が整います。

ウィザードの各手順
------------------

以下の 5 ステップが順番に表示されます。画面の指示に従って進めてください。

#. ノート環境の初期化  

   _project や _templates/note ディレクトリを作成し、必要なテンプレートを生成します。

#. コアプラグインの有効化  

   必須のコアプラグインを有効化します。（Outline, Tag Pane, Daily Notes 等）

#. カスタムプラグインのインストールと有効化  

   Tag Wrangler, Calendar などをインストールして有効化します。

#. テーマのセットアップ  

   Minimal テーマをインストールし有効化します。

#. デイリーノートの設定  

   保存先ディレクトリとテンプレートを設定します。

#. Web Clipper の確認  

   ブラウザ拡張 **Obsidian Web Clipper** を新ストールします。  

   - インストールページ: https://obsidian.md/clipper

   Web Clipper をインストールし、実際に一度クリップを実行し ``Clippings/`` フォルダに Web クリップが
   記録されることを確認します。  
   Clippings フォルダが見つからない場合は、Web Clipper が未導入またはまだ利用されていないと判断されます。
